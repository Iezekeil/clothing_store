## 🔴 Критические проблемы

- [x] **#1 — Токен Telegram в открытом коде**
  - В `js/script.js` (строка ~289) захардкожены `TOKEN` и `CHAT_ID`
  - Любой пользователь может увидеть токен через DevTools и злоупотребить ботом
  - *Решение:* Перенести токен на серверную сторону (backend/proxy), либо хотя бы задокументировать риск

- [x] **#2 — `product.html` является нерабочей заглушкой**
  - Страница содержит плейсхолдеры: `via.placeholder.com`, "Название товара", "10 000 ₽"
  - Ни одна карточка в каталоге на неё не ссылается — страница недостижима
  - *Решение:* Либо наполнить реальным контентом и подключить к карточкам, либо убрать файл

## 🟡 Функциональные недоработки

- [x] **#3 — Фильтры "Цена" не работают**
  - В сайдбаре каталога есть поля "от" и "до", но в `script.js` нет логики фильтрации
  - *Решение:* Написать функцию, которая скрывает карточки вне заданного диапазона цен при нажатии "Применить"

- [x] **#4 — Кнопка "Применить" в сайдбаре — декоративная**
  - У `.sidebar__button` нет обработчика события
  - *Решение:* Добавить `addEventListener('click', ...)` и применять все активные фильтры (цена + размер)

- [x] **#5 — Фильтр по размерам не работает**
  - Чекбоксы размеров (XS, S, M, L, XL) в сайдбаре не влияют на отображение товаров
  - *Решение:* Добавить атрибут `data-sizes` на карточки и фильтровать по выбранным чекбоксам

- [x] **#6 — Промокод — нерабочее поле**
  - Поле "Промокод" в `cart.html` есть, но никакой логики нет
  - *Решение:* Добавить список промокодов и логику применения скидки; либо убрать поле, если функционал не нужен

- [x] **#7 — Нет `<span class="error-message">` у поля ФИО**
  - В `cart.html` у `#fio` нет тега для вывода ошибки, в отличие от телефона и email
  - Функция `updateError()` в JS молча не находит элемент и ошибка не отображается
  - *Решение:* Добавить `<span class="error-message" data-for="fio"></span>` после `#fio`

## 🟢 UX и дизайн

- [x] **#8 — Шапка не фиксируется при прокрутке**
  - В длинном каталоге шапка уходит вверх, пользователь теряет навигацию
  - *Решение:* Добавить в CSS: `.header { position: sticky; top: 0; z-index: 100; }`

- [x] **#9 — Нет бургер-меню на мобильных устройствах**
  - На экранах <768px навигация полностью скрыта (`display: none`) без замены
  - Пользователь на телефоне не может перейти в каталог через шапку
  - *Решение:* Добавить кнопку-гамбургер и выдвижное меню на мобильных

- [x] **#10 — Нет активного состояния у ссылок в навигации**
  - Текущая страница никак не выделена в шапке (нет active-класса)
  - *Решение:* В JS определять текущий URL и добавлять CSS-класс `.active` к нужному `nav__link`

- [x] **#11 — Базовый шрифт Arial**
  - `font-family: 'Arial', sans-serif` — слишком стандартно для fashion-бренда
  - *Решение:* Подключить Google Font (например, **Inter**, **Outfit** или **Montserrat**) и обновить `font-family` в CSS

- [x] **#12 — Некорректный путь к hero-изображению**
  - В `css/style.css` прописан `url('logo_main.png')`, но файл лежит в папке `css/`
  - CSS уже находится в `css/`, поэтому путь относительно CSS-файла — верный (`logo_main.png`)
  - Однако папка называется `css/` — лучше вынести изображения в отдельную папку `images/`
  - *Решение:* Создать папку `images/`, переместить `logo_main.png` туда, обновить путь в CSS

## 💡 Дополнительные идеи (бонус)

- [ ] **#13 — Дублирование шапки и футера**
  - Шапка и футер скопированы в 4 HTML-файлах: при изменении нужно обновлять везде
  - *Решение:* Использовать JS-включения (`fetch` + `innerHTML`) или шаблонизатор

- [x] **#14 — Анимация появления карточек в каталоге**
  - Карточки появляются мгновенно — нет плавного эффекта при загрузке страницы
  - *Решение:* Добавить CSS-анимацию `fadeInUp` с задержкой через `animation-delay` на каждую карточку

- [x] **#15 — Skeleton-загрузка изображений**
  - При медленном соединении изображения товаров грузятся без заглушки
  - *Решение:* Добавить серый placeholder (`background: #f0f0f0`) через CSS пока `img` не загрузилось

- [x] **#16 — Счётчик товаров отображает "0" при загрузке**
  - В HTML шапки прописан `<span>0</span>`, а JS обновляет его позже
  - При быстром рендере на долю секунды виден "0"
  - *Решение:* Изначально оставить span пустым `<span></span>` (CSS `.header__cart-count:empty` уже скрывает его)
